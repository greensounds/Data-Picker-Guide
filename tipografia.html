Tipografía para la visualización de datos

Un problema común al momento de realizar una visualización de datos es elegir incorrectamente la tipografía. Al elegir una tipografía que no es adecuada surgen algunos problemas para el usuario. Por ejemplo un error común es elegir una tipografía que sea de difícil legibilidad y el usuario peredera interes en el gráfico. Otro problema común es no ser consistentes con el tamaño de la fuente que elegimos.

Después de analizar a tres autores que hablan sobre tipografía para la visualización de datos (Miko Yuk y Stephanie Diamond, Howard Coale, y Amy cesal)), se propone esta serie de recomendaciones para seleccionar tipografía.

Recomendaciones

  • Utilizar diferentes tamaños de fuente

    • Esta punto se refiere a utilizar un tamaño para los títulos de la visualización, otro diferente para la información de los ejes, otro diferente para la información complementaria, etc.

  • Ser consistentes con los tamaños de fuente

    • Esta punto se refiere a que se debe respetar el tamañano de fuente que se ha elegido para los diferentes puntos de la visualización. Por ejemplo si la información de los ejes tiene un tamaños de 14px, todos los ejes deben tener el mismo tamaño de fuente.

  • Evitar tipografías personalizadas en la visualización

    • Este punto se refiere a evitar utilizar tipografías poco comunes para el lector. Utilizar tipografías poco comunes puede causar que el lector se sienta incómodo al momento de leer la información, o puede distraer la atención del lector provocando que éste no se enfoque hacía el gráfico sino hacía la tipografía.

  • Tipografías recomendadas

    • Algunas de las tipografías que se recomienda utilizar para la visualización de datos son: Verdana, Garamond, Arial, Helvetica, Gill Sans, Itc Franlin Gothic(Como observación la mayoría de las tipografías que se recomiendan son de la familia sans-serif, excepto la tipografía Garamond que es de la familia serif)
